# COCOMO

Оценка выполнена приближенно, потому что проект учебный и включает backend, Android-клиент и документацию.

| Показатель | Значение |
| --- | --- |
| Тип проекта | Organic |
| Оценочный объем | около 5 KLOC |
| Коэффициенты | a = 2.4, b = 1.05 |

Формула трудоемкости:

```text
PM = a * KLOC^b
```

Оценка:

```text
PM = 2.4 * 5^1.05 ≈ 13 человеко-месяцев
```

Для курсового проекта это значение используется как учебная модель. Фактическая разработка велась одним студентом и была ограничена сроками семестра.

## Интерпретация результата

Полученное значение не означает, что проект реально выполнялся командой в течение 13 календарных месяцев. COCOMO используется как приближенная модель оценки трудоемкости программного продукта по объему кода. Для учебного проекта результат показывает относительную сложность системы: в нее входят мобильный клиент, серверная часть, база данных, авторизация, оффлайн-кэш, тестирование и документация.

Фактическое планирование работ отражено в [WBS](wbs.md) и [диаграмме Ганта](gantt.md). Поэтому COCOMO следует рассматривать как расчетную оценку, а диаграмму Ганта - как календарное представление выполнения этапов.
