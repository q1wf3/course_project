# Пояснительная записка

## Тема

Разработка мобильного приложения для ведения коллекции фильмов.

## Введение

Актуальность проекта связана с тем, что пользователи регулярно ведут списки просмотренных и запланированных фильмов, но часто делают это в неподходящих инструментах: заметках, таблицах или сообщениях. Это усложняет поиск, фильтрацию, фиксацию оценок и анализ личной коллекции. Разрабатываемая система решает задачу централизованного учета фильмов через мобильное приложение.

Цель работы — спроектировать и разработать мобильное приложение для ведения коллекции фильмов с серверной частью, REST API, безопасной аутентификацией и оффлайн-кэшированием.

## 1. Аналитическая часть

Предметная область включает хранение фильмов, жанров, пользовательских коллекций, статусов просмотра, оценок и заметок. Основная проблема текущего процесса — разрозненность данных и отсутствие удобного поиска.

Бизнес-анализ, SWOT, стейкхолдеры и бизнес-модель приведены в `docs/00-project-charter/project-charter.md`.

## 2. Проектная часть

Системные требования включают регистрацию, вход, CRUD фильмов, поиск, фильтрацию, управление статусами просмотра и оффлайн-режим. Модель требований приведена в `docs/01-requirements/requirements.md`.

Архитектура построена по PCMEF. Клиентская часть содержит Presentation и Control для UI, серверная часть содержит Control, Mediator, Entity и Foundation. Архитектурное описание приведено в `docs/02-architecture/architecture.md`.

Проектирование базы данных выполнено до третьей нормальной формы. Основные таблицы: `users`, `movies`, `genres`, `collection_items`, `reviews`, `tags`. DDL приведен в `docs/03-database/database.md`.

## 3. Реализационная часть

Серверная часть реализуется на Java Spring Boot. Мобильный клиент реализуется на Kotlin с Jetpack Compose. Для сетевого обмена используется Retrofit, для локального кэша — Room. Описание реализации приведено в `docs/05-implementation/implementation.md`.

## 4. Тестирование и качество

Качество обеспечивается модульными тестами сервисов, интеграционными тестами REST API и ручным тестированием мобильного интерфейса. План покрытия и тест-кейсы приведены в `docs/06-testing/testing.md`.

## 5. Развертывание

Сервер запускается как Spring Boot-приложение с подключением к PostgreSQL. Клиент собирается в APK через Android Studio или Gradle. Инструкция приведена в `docs/10-deployment/deployment.md`.

## 6. Управление проектом

Проект выполняется по 18-недельному плану:

| Недели | Этап |
|---|---|
| 1–2 | Инициация и бизнес-анализ |
| 3–4 | Требования |
| 5–6 | Архитектура |
| 7–8 | База данных |
| 9–10 | Детальное проектирование |
| 11–12 | Реализация ядра |
| 13–14 | Рефакторинг и качество |
| 15–16 | Интерфейс |
| 17–18 | Завершение и защита |

## Заключение

В результате проектирования сформирована архитектура мобильного приложения для ведения коллекции фильмов, описаны требования, база данных, REST API, интерфейс, тестирование и развертывание. Проект соответствует мобильной траектории и требованиям PCMEF.
