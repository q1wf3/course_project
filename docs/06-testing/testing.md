# Этап 6. Тестирование

## Стратегия тестирования

| Уровень | Инструмент | Что проверяется |
|---|---|---|
| Unit | JUnit 5, Mockito | Бизнес-логика сервисов |
| Integration | Spring Boot Test | REST API и репозитории |
| UI | Android instrumented tests | Навигация и состояния экранов |
| API | Postman/Swagger | Контракты endpoints |

## Тест-кейсы

| ID | Проверка | Ожидаемый результат |
|---|---|---|
| TC-01 | Регистрация с новым email | Пользователь создан, возвращен JWT |
| TC-02 | Регистрация с существующим email | Ошибка 409 |
| TC-03 | Добавление фильма | Фильм появляется в коллекции |
| TC-04 | Повторное добавление фильма | Ошибка бизнес-правила |
| TC-05 | Поиск по названию | Возвращаются подходящие фильмы |
| TC-06 | Оффлайн-режим | Данные отображаются из Room-кэша |
| TC-07 | Удаление фильма | Запись исчезает из коллекции |

## План покрытия

| Модуль | Минимальное покрытие |
|---|---|
| `MovieService` | 60% |
| `AuthService` | 50% |
| `JwtService` | 80% |
| `Movie` и `CollectionItem` | 80% |
| Общий backend | >40% |

## Реализованные unit-тесты

| Тест | Проверка |
|---|---|
| `CollectionItemTest` | изменение статуса, валидация оценки |
| `MovieTest` | нормализация категории, валидация названия и года выпуска |
| `JwtServiceTest` | генерация JWT, чтение userId, отказ при подмене подписи |
| `MovieServiceImplTest` | создание фильма, получение фильма по ID из коллекции пользователя |

## Команда для отчета JaCoCo

```bash
mvn test jacoco:report
```
