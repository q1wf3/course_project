# Identity Map

Identity Map в проекте обеспечивается Hibernate Persistence Context во время транзакции. Если одна и та же сущность загружается повторно по идентификатору в рамках одного контекста, Hibernate возвращает тот же объект.

В проекте это важно для сущностей:

- `User`;
- `Movie`;
- `CollectionItem`.

Практический эффект:

- меньше дублирующих SQL-запросов;
- согласованное состояние сущностей внутри операции сервиса;
- корректное сохранение изменений через dirty checking.
