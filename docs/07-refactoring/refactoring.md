# Этап 7. Рефакторинг и качество

## Data Mapper

Data Mapper используется для отделения JPA-сущностей от DTO. Контроллеры не получают и не возвращают сущности напрямую, поэтому REST API не зависит от внутренней структуры базы данных.

Пример:

```java
public MovieDto toDto(Movie movie) {
    return new MovieDto(movie.getId(), movie.getTitle(), movie.getReleaseYear());
}
```

## Identity Map

Identity Map применяется на уровне транзакции через persistence context JPA. Это гарантирует, что в рамках одной транзакции одна запись базы данных представлена одним объектом, а повторные обращения не создают конфликтующие экземпляры.

## Lazy Load

Связи `Movie -> Genre` и `CollectionItem -> Review` загружаются отложенно. Это уменьшает нагрузку при отображении списка фильмов, где не всегда нужны полные детали.

## Статический анализ

Для контроля качества используются:

- Checkstyle для стиля кода;
- JaCoCo для покрытия;
- SonarLint или SonarQube для поиска дефектов;
- ручная проверка отсутствия циклических зависимостей между слоями PCMEF.

## Устраненные запахи кода

| Запах | Решение |
|---|---|
| Fat Controller | Бизнес-логика вынесена в `MovieService` |
| Anemic Domain Model | В `CollectionItem` добавлены методы изменения статуса и оценки |
| Data Class DTO leakage | Добавлен `MovieMapper` |
| Дублирование валидации | Общая валидация команд в Control/Mediator |
