# ADR: Архитектурные решения

## ADR-01. Android Native + Jetpack Compose

**Решение:** реализовать клиент как Android-приложение на Kotlin и Jetpack Compose.  
**Причина:** проект относится к мобильной траектории, а Compose позволяет быстро создавать современный интерфейс.

## ADR-02. Spring Boot REST API

**Решение:** реализовать backend на Java 17 и Spring Boot.  
**Причина:** Spring Boot предоставляет готовую инфраструктуру для REST, JPA, Security и тестирования.

## ADR-03. PostgreSQL как основная база данных

**Решение:** хранить серверные данные в PostgreSQL 16.  
**Причина:** методические требования предполагают полноценную СУБД, а Docker Compose делает запуск воспроизводимым.

## ADR-04. Room для оффлайн-кэша

**Решение:** использовать Room на Android для локального хранения последней коллекции.  
**Причина:** пользователь должен видеть данные после выключения приложения или при временной недоступности backend.

## ADR-05. JWT и BCrypt

**Решение:** использовать JWT для запросов и BCrypt для хранения паролей.  
**Причина:** пароль не должен храниться открытым текстом, а API должен проверять права доступа.
