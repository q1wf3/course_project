# BUC-диаграмма

BUC-диаграмма отражает основные бизнес-прецеденты проекта.

```plantuml
@startuml
left to right direction
actor "Гость" as Guest
actor "USER" as User
actor "ADMIN" as Admin

usecase "Регистрация" as UC1
usecase "Вход" as UC2
usecase "Просмотр коллекции" as UC3
usecase "Добавление фильма" as UC4
usecase "Редактирование фильма" as UC5
usecase "Поиск фильма" as UC6
usecase "Работа оффлайн" as UC7
usecase "Просмотр статистики" as UC8
usecase "Управление пользователями" as UC9
usecase "Просмотр коллекций пользователей" as UC10

Guest --> UC1
Guest --> UC2
User --> UC3
User --> UC4
User --> UC5
User --> UC6
User --> UC7
Admin --> UC8
Admin --> UC9
Admin --> UC10
@enduml
```

![BUC-диаграмма](images/buc-diagram.png)

Разделение ролей важно для проекта: обычный пользователь работает только со своей коллекцией, а администратор получает отдельный набор сценариев сопровождения системы.
Гость ограничен регистрацией и входом, потому что до авторизации система не должна открывать личные данные. Пользователь выполняет основные операции с фильмами, а администратор контролирует состояние приложения через статистику, роли и просмотр коллекций.
