# Этап 1. Проектирование требований

## Цель этапа

Описать требования к Movie Collection на уровне пользователя и системы: функциональные возможности, нефункциональные ограничения, варианты использования, модель предметной области и связь требований с реализацией.

| [Диаграмма вариантов использования](use-case-diagram.md) | Сценарии гостя, пользователя и администратора | Выполнено |
| [Спецификации вариантов использования](use-case-specifications.md) | Подробные сценарии входа, регистрации, добавления фильма, поиска и администрирования | Выполнено |
| [Модель предметной области](domain-model.md) | Связи между пользователем, фильмом, записью коллекции, категорией, ролью и статусом | Выполнено |
| [Расширенный глоссарий](expanded-glossary.md) | Уточненные термины требований и предметной области | Выполнено |
