# Спецификации вариантов использования

## UC-01. Регистрация

**Актор:** гость.  
**Предусловие:** пользователь не авторизован.  
**Основной сценарий:** пользователь вводит email и пароль, приложение отправляет запрос `/api/auth/register`, backend создает учетную запись и возвращает JWT.  
**Постусловие:** пользователь авторизован с ролью `USER`.

## UC-02. Добавление фильма

**Актор:** USER.  
**Предусловие:** пользователь авторизован.  
**Основной сценарий:** пользователь открывает экран `+`, вводит название, год, режиссера, категорию, статус и оценку, затем сохраняет карточку.  
**Постусловие:** фильм появляется в коллекции и сохраняется в PostgreSQL.

## UC-03. Оффлайн-просмотр коллекции

**Актор:** USER.  
**Предусловие:** коллекция хотя бы раз была загружена.  
**Основной сценарий:** при недоступном backend приложение показывает данные из Room.  
**Постусловие:** пользователь может просматривать последнюю сохраненную коллекцию.

## UC-04. Управление пользователями

**Актор:** ADMIN.  
**Предусловие:** пользователь вошел с ролью `ADMIN`.  
**Основной сценарий:** администратор открывает панель, просматривает список учетных записей, меняет роль или удаляет пользователя.  
**Постусловие:** изменения применяются на сервере.
